ESP32-S3 On-Device LLM — Training & Converter Package
=======================================================

This tool trains a tiny GPT-2-style language model on your PC and converts it to
a model.bin that runs on the ESP32-S3 (8 MB PSRAM). The model is DOMAIN-AGNOSTIC:
you supply the knowledge. The training/ folder in this repo ships trainers and
validation tools — not example data.

FOUR WAYS TO GET A MODEL
-------------------------

  A) Pre-trained model (fastest)
     Download a ready .bin from:
       Training Material + Pre-trained Models/Trained + Ready Models/
     Copy to /sd/llm/ on the device. No training step.

  B) Your own dataset
     Write a data .txt file, train with the scripts in this training/ folder.

  C) Example training package
     Unzip a package from:
       Training Material + Pre-trained Models/Training Materials/
     Each zip is self-contained (trainers + training_data/ + scripts + instructions).
     Unzip anywhere, cd into it, and train from there.

  D) Build your own on ANY topic, AI-assisted (easiest custom model)
     Use the template kit at:
       Training Material + Pre-trained Models/Training Materials/build_your_own_model/
     Hand the folder to an AI (Claude, ChatGPT, ...) with any topic you want. Per
     that folder's BUILD_YOUR_OWN_MODEL.md, the AI fills in a generator that emits
     a full, correct training corpus for the topic — facts under many phrasings,
     reverse/aggregate lookups, precomputed reasoning, and lore. The kit
     generates the data only; once the corpus is written, train it with the
     trainer in this training/ folder (STEP 3). No hand-writing a dataset
     (Path B) and no waiting for a shipped package (Path C) — just describe the
     topic.


STEP 1 — Install dependencies (once)
--------------------------------------

For Path B, run from this training/ folder. For Path C, run from inside the
unzipped package (it includes its own copy of requirements.txt).

    pip install -r requirements.txt

On GPU machines, install PyTorch for your CUDA version first:

    CUDA 12.1:  pip install torch --index-url https://download.pytorch.org/whl/cu121
    CUDA 12.4:  pip install torch --index-url https://download.pytorch.org/whl/cu124

Verify GPU (optional):  python -c "import torch; print(torch.cuda.is_available())"


STEP 2 — Get your training data (Path B or C)
-----------------------------------------------

PATH B — Roll your own data
  Write a plain .txt file of blocks separated by blank lines. Three block types:

      Q: <question>            Q: <question>           <a prose paragraph of
      A: <short answer>        Do: <command>            background context>

  Tips:
    - Keep answers short (~<30 words) so a full Q&A pair fits the context window.
    - Reinforce each fact with several phrasings of the question -> same answer.
    - Use Q:/Do: only if your agent suggests named commands; otherwise skip it.
    - If your domain has multi-word terms or commands that must tokenize as one
      unit, list them one-per-line in a file and pass --special-tokens that_file.

  Need a format reference? Each training package zip includes training_data/
  template.txt — unzip any package and copy that file as a starting point.

PATH C — Use a shipped example package
  1. Pick a zip from Training Material + Pre-trained Models/Training Materials/
     (HardwareOne Help Agent, Kanto Pokemon Master, or Periodic Table Guide).
  2. Unzip it, e.g.:
         unzip hardwareone_training_package.zip -d my_model && cd my_model
  3. The package already contains training_data/, trainers, and validation scripts.
     Follow the train commands in that package's INSTRUCTIONS.txt or the catalog
     README at Training Material + Pre-trained Models/Training Materials/README.md.

PATH A — Skip this step entirely if you only want a pre-trained .bin.


STEP 3 — Train the model (Path B or C)
----------------------------------------

Use HW1HelpAgent192_deep for best quality: 16 layers, dim=192, 6 heads, FFN=512,
seq=128. (The preset names are historical labels; they work for any domain.)

PATH B example (your own files, run from training/):

    python train_tiny_model_gpu.py \
        --preset HW1HelpAgent192_deep \
        --text /path/to/YOUR_DATA.txt \
        --qa-test-prompts /path/to/YOUR_TEST_PROMPTS.txt \
        --epochs 250 --lr 3e-4 --batch-size 16 \
        --out ./out_mymodel

PATH C example (unzipped HardwareOne package, run from inside the unzip dir):

    python train_tiny_model_gpu.py \
        --preset HW1HelpAgent192_deep \
        --text training_data/hardwareone_rich.txt \
        --special-tokens training_data/hardwareone_special_tokens.txt \
        --epochs 250 --lr 3e-4 --batch-size 16 \
        --out ./out_HW1HelpAgent192_deep

CPU training (slow, use only if no GPU available):

    python train_tiny_model.py \
        --preset HW1HelpAgent192_deep \
        --text <your-data.txt> \
        --epochs 400 --batch-size 8 --lr 3e-4 \
        --out ./out_mymodel

Add  --special-tokens <file>  if your domain needs whole-word tokens.

Loss should fall from ~7 down to ~0.1-0.3. After training, the script runs a
generation test. Pass --qa-test-prompts (a file of "Q: ..." lines) so the test
asks questions from YOUR domain; otherwise it uses a generic placeholder.

After saving the model, both trainers auto-extract a domain word-list from your
corpus and write it as domain_vocab.txt into the --out folder. This drives the
on-device refusal gate (see STEP 4): a prompt that mentions none of your domain's
words gets a fixed refusal instead of a hallucinated answer. Pass
--no-domain-vocab to skip writing the list.

To check model size before training:

    python train_tiny_model_gpu.py --preset HW1HelpAgent --estimate-only


STEP 4 — Convert to model.bin (browser tool)
---------------------------------------------

4a. Open index.html in Chrome or Edge (Firefox works too).
    No server needed — open the file directly from disk (repo root).

4b. Drop the entire output folder (./out_mymodel) onto the page, or use
    the file picker to select the files inside it.

4c. Select INT8 quantization, group size 128. These are the defaults.

4d. Domain refusal gate (optional). If the dropped folder contains a
    domain_vocab.txt, the converter auto-loads it into the "Domain words" field;
    you can also pick a list with the manual "Domain words" file input and set a
    "Refusal answer". With both set, the device refuses any prompt that contains
    none of the domain words, replying with the Refusal answer instead of
    guessing. Clear both fields to disable the gate.

4e. Click Convert. When finished, click Download. This saves model.bin.


STEP 5 — Copy to device
-------------------------

5a. Copy model.bin to the SD card at /sd/llm/ or upload via the web Files page.

5b. Open the LLM tab in the web dashboard, select the model, and click Load.
    Or from the CLI: llm load /sd/llm/model.bin

5c. Expected log output: [LLM] Model ready. PSRAM used: ~6900KB


STEP 6 — Use the LLM
----------------------

Ask questions from the LLM tab in the web dashboard or from the CLI:

    llm generate "your question here"

The firmware stops generation after 2 sentences or 80 tokens (configurable).


WHAT LIVES WHERE
-----------------

  training/  (this repo)
      Trainers, requirements, and validation scripts. No example data.
      Use Path B: point --text at your own .txt file.

  Training Material + Pre-trained Models/Training Materials/
      Self-contained .zip packages (Path C). Each unzip dir has:
          training_data/     data files + template.txt + special-tokens files
          train_tiny_model*.py
          training_scripts/  validators and analysis tools
          INSTRUCTIONS.txt

      build_your_own_model/  Template kit (Path D). Hand it to an AI with a topic
          and it writes the whole data generator for you; it emits the corpus only,
          then train it with the trainer in this training/ folder. Start at its
          BUILD_YOUR_OWN_MODEL.md.

  Training Material + Pre-trained Models/Trained + Ready Models/
      Pre-trained model.bin files (Path A). Copy straight to the device.

  training_scripts/  (in this repo)
      run_all_checks.py                 Run all data quality checks at once
      deep_error_analysis.py            12-check structural and content analysis
      validate_training_data.py         Format and command validation
      topic_coverage_report.py          Count QA pairs per topic
      extract_domain_vocab.py           Build or tune a domain word-list by hand
      ... (plus other analysis tools)

Note: The GPU trainer masks question tokens during training (only answer tokens
contribute to the loss). This focuses learning on answer quality.
