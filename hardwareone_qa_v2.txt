Q: How do I connect to WiFi?
A: WiFi connection is configured through Settings. Open the web dashboard, go to Settings, enter your WiFi SSID and password, then save. The device reconnects automatically on reboot.

Q: How do I set up WiFi on Hardware One?
A: WiFi credentials are stored in Settings. Navigate to the Settings page on the dashboard, fill in the WiFi SSID and password fields, and tap Save.

Q: Where do I enter my WiFi password?
A: WiFi password is entered in the Settings page of the web dashboard. Go to Settings, find the WiFi section, type your network name and password, and save.

Q: Can Hardware One connect to a wireless network?
A: WiFi is supported natively. Use the Settings page to enter your network SSID and password. After saving, the device connects and its IP address appears on the OLED display.

Q: My device won't connect to my network. What should I check?
A: WiFi connection issues are usually caused by a wrong SSID or password. Verify the credentials in Settings and confirm the network is 2.4 GHz since the ESP32-S3 does not support 5 GHz.

Q: What is ESP-NOW and how is it different from WiFi?
A: ESP-NOW is a peer-to-peer radio protocol for direct device-to-device communication. It works without any router or access point infrastructure.

Q: How do I send a message to another Hardware One over ESP-NOW?
A: ESP-NOW messaging is done through the espnow command in the CLI. First pair the two devices using the dashboard ESP-NOW peer panel, then use espnow send followed by the peer name and message text.

Q: How do I pair two Hardware One devices over ESP-NOW?
A: ESP-NOW pairing is done from the web dashboard. Open the ESP-NOW section, click Add Peer, and enter the MAC address of the other device. Both units must add each other as peers.

Q: Can Hardware One communicate without a router?
A: ESP-NOW allows direct device-to-device communication with no router required. Two Hardware One units can exchange messages and sensor data peer-to-peer.

Q: What range does ESP-NOW have?
A: ESP-NOW range is approximately 50 to 100 meters in open air. It uses the 2.4 GHz radio and establishes a direct encrypted link between paired device MAC addresses.

Q: How does bonded device mode work?
A: ESP-NOW bonded mode creates a trusted relationship between two Hardware One units. Once bonded, the CLI on one device can transparently run commands on the other without any extra syntax.

Q: Can I run a command on a remote Hardware One?
A: ESP-NOW bonded device mode lets you run commands on a paired unit. After bonding two devices via the dashboard, any CLI command you type can be forwarded to and executed on the bonded partner.

Q: How do I enable Bluetooth on Hardware One?
A: Bluetooth is enabled through the feature flags in Settings. Go to Settings, find the BLE toggle, enable it, and save. The device will begin advertising over Bluetooth Low Energy.

Q: What does BLE do on this device?
A: Bluetooth Low Energy enables proximity interactions and short-range communication. Hardware One uses BLE to broadcast a beacon, detect nearby devices, and optionally exchange small data payloads.

Q: How do I connect to an MQTT broker?
A: MQTT is configured in Settings. Enter your broker hostname or IP address, port number (typically 1883), and optionally a username and password. After saving, Hardware One connects and subscribes to its configured topics.

Q: What MQTT topics does Hardware One publish to?
A: MQTT topic names are set in Settings. By default the device publishes sensor readings and status updates to topics under the device name prefix.

Q: How do I trigger an MQTT message from automation?
A: MQTT publish is one of the available automation actions. In the Automation page, create a rule, add a Publish MQTT action, and fill in the topic and payload.

Q: How do I perform an OTA firmware update?
A: OTA update is done from the web dashboard. Go to the OTA section, select a firmware binary (.bin file), and click Upload. The device flashes the new image and reboots into the new firmware.

Q: Can I update the firmware without a USB cable?
A: OTA over-the-air updates are supported. Open the web dashboard, navigate to the OTA page, upload the new firmware binary, and Hardware One will flash and reboot automatically.

Q: What happens if an OTA update fails?
A: OTA rollback is automatic. Hardware One uses a dual-partition scheme, so if the new firmware fails to boot, the device falls back to the previous working image.

Q: How do I read temperature and humidity?
A: The BME280 sensor measures temperature and humidity. If connected on the I2C bus, the device detects it automatically and shows readings on the dashboard and OLED.

Q: What does the BME280 measure?
A: The BME280 measures temperature, humidity, and barometric pressure. It connects via I2C and is auto-detected at startup. Readings appear on the web dashboard.

Q: How do I read motion data?
A: The ICM-42688-P IMU provides motion and orientation data. This sensor is auto-detected on the I2C bus and provides accelerometer and gyroscope readings on the dashboard.

Q: What is the IMU used for?
A: The ICM-42688-P inertial measurement unit provides accelerometer and gyroscope data for motion and orientation sensing. Use it to detect tilt, vibration, or movement.

Q: How do I measure distance with the device?
A: The VL53L1X time-of-flight sensor measures distance. Connect it to the I2C bus and it will be detected automatically. The sensor outputs distance in millimeters.

Q: What sensor is used for distance?
A: The VL53L1X is the time-of-flight distance sensor. It measures distances up to 4 meters using infrared light and connects over I2C.

Q: How do I measure surface temperature without touching it?
A: The MLX90614 infrared thermometer measures non-contact surface temperature. Connect it to the I2C bus and the device will detect it automatically.

Q: Can I detect gestures with Hardware One?
A: The APDS9960 sensor detects gestures. It recognizes up, down, left, and right swipes, along with proximity and ambient light level, over I2C.

Q: What does the APDS9960 detect?
A: The APDS9960 detects gestures, proximity, and ambient light. It recognizes directional swipes (up, down, left, right) and can trigger automation rules based on gesture events.

Q: How do I get GPS coordinates?
A: GPS coordinates come from the GPS module connected over UART. The device parses NMEA sentences and tracks position fix status. Coordinates are displayed on the dashboard map tab.

Q: How do I view my position on a map?
A: GPS position is shown on the Map tab of the web dashboard. The map renders using tiles cached locally on the SD card, so it works offline in the field.

Q: Does GPS work without internet?
A: GPS works fully offline. Map tiles are pre-cached on the SD card, and the GPS module uses satellite signals, not internet connectivity.

Q: How do I control servos?
A: The PCA9685 PWM driver controls servos. Connect it to the I2C bus and it will be auto-detected. Use the servo command in the CLI to set channel and position in degrees.

Q: How do I control PWM outputs?
A: The PCA9685 provides 16 PWM channels controllable from the CLI using the servo or pwm commands. It is I2C-based and detected automatically at startup.

Q: How do I control NeoPixel LEDs?
A: NeoPixel control uses the neopixel command in the CLI. Specify the number of LEDs, the GPIO data pin, a color as RGB hex, and optional brightness.

Q: How do I change the LED strip color?
A: NeoPixel LED color is set with the neopixel command. Use neopixel set followed by the LED count, GPIO pin, color in hex, and brightness level.

Q: How do I browse files on the device?
A: Files are browsed through the Files tab on the web dashboard. You can see files on both the internal LittleFS filesystem and the SD card, and download, upload, or delete files.

Q: How do I upload a file to Hardware One?
A: Files are uploaded on the Files tab of the web dashboard. Click the upload button, select a file from your computer, and it will be transferred to the device.

Q: What is LittleFS?
A: The filesystem LittleFS is the internal persistent storage on Hardware One. It stores settings and configuration files in flash memory and survives power cycles and OTA updates.

Q: What is the SD card used for?
A: The SD card stores large files that do not fit in flash, including LLM model files, map tiles for GPS, and log files.

Q: How do I set up an automation rule?
A: Automations are created on the Automation page of the dashboard. Click Add Rule, choose a condition type (sensor threshold, time trigger, or received message), then add one or more actions.

Q: Can Hardware One react to sensor readings automatically?
A: Automations respond to sensor values without manual input. Create a rule on the Automation page, set a condition like BME280 temperature above 30, and add an action.

Q: How do I combine multiple conditions in an automation rule?
A: Automations support AND and OR operators between conditions. Select the logical operator when adding conditions to a rule. The rule only fires when the combined condition is true.

Q: How do I use the LLM on this device?
A: The language model is accessed through the LLM tab on the web dashboard. Type your prompt and submit it. The model generates a response streamed live to the browser.

Q: How do I load a language model?
A: The language model is loaded by placing a model.bin file in LLM1 format on the SD card. Use the llm load command in the CLI or select it from the LLM tab on the dashboard.

Q: What model format does Hardware One use?
A: The language model uses LLM1 binary format. It stores transformer weights in either FP32 or INT8 quantized form. The converter tool generates LLM1 files from standard checkpoints.

Q: How do I convert a model for the device?
A: The language model converter tool at index.html handles conversion. Drag in the model output folder, select INT8 quantization, and download the model.bin file for the SD card.

Q: How do I run the LLM from the command line?
A: The language model CLI command is llm generate followed by your prompt in quotes. The model generates a streamed response directly in the terminal.

Q: How do I see what I2C devices are connected?
A: I2C scanning is done with the i2c scan command in the CLI. It returns detected device addresses on the bus. Hardware One also scans automatically at startup.

Q: Does Hardware One auto-detect connected sensors?
A: I2C sensor auto-detection runs at startup. Hardware One scans the bus and automatically loads drivers for recognized devices including BME280, ICM-42688-P, VL53L1X, MLX90614, APDS9960, and PCA9685.

Q: How do I see sensor readings in real time?
A: Hardware One shows real-time sensor data on the Sensors tab of the web dashboard. Values update automatically. You can also use the sensor command in the CLI.

Q: How do I use the OLED display?
A: The OLED display shows status information automatically. It displays the IP address, GPS status, and sensor summaries. You can write custom text with the display command in the CLI.

Q: What does the OLED show by default?
A: The OLED shows the device IP address, connection status, GPS fix state, and a rotating summary of sensor readings. It updates automatically.

Q: How do I use the gamepad?
A: The Adafruit Seesaw gamepad connects over I2C and is auto-detected at startup. Button presses and joystick position can trigger automation rules or be queried with the gamepad command.

Q: How do I back up my settings?
A: Settings export is available from the Settings page of the dashboard. Click Export to download a JSON file containing all your configuration.

Q: How do I restore settings to another device?
A: Settings import is done from the Settings page. Click Import, select the previously exported JSON file, and the configuration is applied immediately.

Q: How do I view the serial logs?
A: The CLI serial output is available from the web terminal on the dashboard as well as over the physical UART port. Tagged log output can also be saved to a file on the SD card.

Q: Can I save log output to a file?
A: The CLI log file command enables debug log output to a file. Specify a filename on the SD card and tagged log output will be written there for post-mortem analysis.

Q: How do I reset the device to factory defaults?
A: Hardware One factory reset is done through the Settings page. Click Reset to Defaults to wipe all stored settings. The device reboots into unconfigured state.

Q: What is the web terminal?
A: The CLI web terminal is accessible from the Terminal tab on the dashboard. It accepts the same commands as the physical serial port without needing a USB cable.

Q: How do I register a custom CLI command?
A: The CLI register API in firmware lets you add custom commands at runtime. Provide a command name, handler function, and help string.

Q: What communication protocols does Hardware One support?
A: Hardware One supports multiple independent communication protocols for different use cases. Check the settings for each protocol to configure them individually.

Q: What is the difference between ESP-NOW and MQTT?
A: ESP-NOW is a direct peer-to-peer radio link between two Hardware One units with no infrastructure needed. It is best for local device-to-device communication.

Q: What is the difference between MQTT and WiFi?
A: MQTT is an application-layer messaging protocol that runs on top of the network connection to exchange messages with a broker. It handles publish-subscribe messaging for sensor data and control commands.

Q: What processors does Hardware One use?
A: Hardware One uses the ESP32-S3 microcontroller with a dual-core Xtensa LX7 processor running at up to 240 MHz and 8 MB of PSRAM.

Q: How much memory does Hardware One have?
A: Hardware One has 8 MB of PSRAM for large runtime allocations like LLM weights, plus 16 MB of flash storage and the standard ESP32-S3 internal SRAM.

Q: What is PSRAM used for?
A: Hardware One uses PSRAM to store large data that does not fit in internal RAM, including LLM model weights, large string buffers, and sensor history tables.

Q: How does the authentication work?
A: Hardware One authentication uses a password-based login. You set a password in Settings and the device issues a session token on successful login. Auth is fully on-device.

Q: Is there a cloud service required?
A: Hardware One operates entirely standalone with no cloud service required. The web dashboard is served from the device, LLM inference runs locally, and map tiles are cached on the SD card.

Q: How do I access the web dashboard?
A: Hardware One web dashboard is accessed by opening a browser and navigating to the device IP address shown on the OLED display. Login with the password set in Settings.

Q: What tabs does the web dashboard have?
A: Hardware One web dashboard includes tabs for Sensors, Map, Files, Terminal, Automation, LLM, ESP-NOW, OTA, and Settings.

Q: How are FreeRTOS tasks organized?
A: Hardware One runs each major subsystem in its own FreeRTOS task with configured stack sizes and priorities. This prevents any one subsystem from blocking others.

Q: How does the build system work?
A: The build system uses ESP-IDF. Run idf.py build to compile, idf.py flash to write over USB, and idf.py monitor to see serial output.

Q: What is the partition table layout?
A: The build system partition table includes two application partitions for OTA, a LittleFS data partition, and a NVS partition. Two app partitions allow rollback if needed.

Q: How do I train a new LLM for this device?
A: The language model is trained using the Python scripts included in the toolchain. Run train_tiny_model_gpu.py with your text files, then use the converter to package the output into LLM1 format.

Q: What model size fits on Hardware One?
A: The language model size is limited by the 8 MB PSRAM. The narrow preset produces around 1 to 2 MB. The stretch2 preset approaches 8 MB. Models must be INT8 quantized for larger presets.

Q: What is INT8 quantization?
A: The language model uses INT8 quantization to reduce weight precision from 32-bit floats to 8-bit integers with grouped scaling factors, shrinking the model to roughly one quarter of FP32 size.

Q: How do I know if my model loaded correctly?
A: The language model shows a READY status in the LLM tab when loaded correctly. Use llm info in the CLI to display architecture type, layer count, embedding dimension, and vocabulary size.

Q: How do I see what model is currently loaded?
A: The language model info is displayed with the llm info command in the CLI. It shows file path, architecture, number of layers, embedding dimension, vocabulary size, and quantization format.
