Q: What is the on-device language model?
A: The LLM is a small GPT-2 language model that runs directly on the ESP32-S3. It uses PSRAM for weights and runs entirely offline.

Q: How do I load a language model?
A: The LLM loads automatically when a model.bin file is placed in /system/llm/ on internal flash or /sd/llm/ on the SD card. The file is produced by the esp32-llm-converter tool.

Q: How do I start the language model?
A: The LLM interface is accessed by typing llm in the CLI. You can also use the LLM tab in the web UI, which shows a chat interface with token-per-second stats.

Q: What model format does Hardware One use?
A: The LLM uses the LLM1 binary format. It embeds the tokenizer, model weights, and configuration in a single file. INT8 quantised models are recommended to fit within the 8 MB PSRAM budget.

Q: How fast does the language model run?
A: The LLM speed depends on the model and quantisation. An INT8 model with 18 layers typically runs at one to three tokens per second on the ESP32-S3.

Q: How do I convert a model for Hardware One?
A: The LLM converter is the esp32-llm-converter web tool. Open index.html, drag your trained model folder in, select INT8 output, and download the resulting model.bin. Upload this file to /system/llm/ or /sd/llm/.

Q: Can the language model run without WiFi?
A: The LLM runs entirely on-device with no network connection required. It uses only local PSRAM and flash storage.

Q: What language model architectures are supported?
A: The LLM supports GPT-2 compatible models and Llama-style models. GPT-2 uses absolute positional embeddings and LayerNorm. Llama uses RoPE and RMSNorm. Both can be INT8 quantised.

Q: How do I stop language model generation?
A: The LLM generation can be stopped by pressing the Stop button in the web UI. From the CLI, press any key or send a stop signal.

Q: What sampling settings does the LLM support?
A: The LLM supports temperature, top-p, and Mirostat v2 sampling. Temperature controls randomness. Top-p filters low-probability tokens. Mirostat targets a specific surprise level per token.

Q: What is the SD card for?
A: The SD card provides extra storage for files, maps, and language models. Files on the SD card are accessible under the /sd/ path.

Q: How do I mount the SD card?
A: The SD card is mounted with the sd mount command. Use sd status to check whether it is mounted and how much space is available.

Q: How do I unmount the SD card?
A: The SD card is safely unmounted with the sd unmount command before removing it.

Q: How do I copy a file to the SD card?
A: Files are copied with the filecopy command followed by source and destination paths. For example, filecopy /system/llm/model.bin /sd/llm/model.bin.

Q: What is the LittleFS filesystem?
A: The filesystem on Hardware One uses LittleFS for internal flash storage. It stores settings, automations, logs, and small files. It is wear-levelled and power-safe.

Q: How do I see the filesystem layout?
A: The filesystem root is listed with the files / command. The main folders are /system for configuration and /data for user files.

Q: What is the I2C dashboard?
A: The I2C dashboard in the web UI shows all detected I2C devices with their addresses and response times. It polls the bus periodically and highlights devices that stop responding.

Q: How do I run the I2C dashboard?
A: I2C devices are shown in the Sensors tab of the web UI under the I2C section. You can also run i2c scan from the CLI for a one-time scan.

Q: What happens if an I2C sensor stops responding?
A: I2C devices that stop responding are marked in red on the dashboard. Use i2c reset to reset the bus without rebooting. If that fails, check wiring or power to the sensor.

Q: How do I see sensor data in real time?
A: The BME280, IMU, and other active sensors show live readings in the Sensors tab of the web UI. You can also use SENSORread from the CLI for a single reading.

Q: How does sensorlog work?
A: The sensor logging system is started with sensorlog start followed by the sensor name. Readings are written to a timestamped file. Use sensorlog stop to stop and sensorlog view to display recent entries.

Q: Where are sensor logs stored?
A: The filesystem stores sensor logs in /data/logs/ on internal flash. Each sensor gets its own file named after the sensor and the start timestamp.

Q: How do I export sensor log data?
A: Files can be viewed with the fileview command followed by the log file path. You can also download log files from the Files tab in the web UI.

Q: What is the offline map feature?
A: GPS tracks can be viewed on an offline tile map in the web UI. Map tiles are stored on the SD card and are fetched automatically when online, then cached for offline use.

Q: How do I use the GPS tracker?
A: GPS tracking starts with the opengps command to activate the PA1010D module. Use gpsread to get a single position fix. The web UI map page shows your current position in real time.

Q: How do I save a GPS track?
A: GPS tracks are saved automatically to /sd/gps/ when the GPS is running. Each session creates a new GPX file. View tracks in the map tab of the web UI.

Q: What is the web dashboard?
A: Hardware One provides a web dashboard as the main overview page. It shows connectivity status, connected sensors, uptime, memory usage, and recent log entries.

Q: How do I access the web dashboard?
A: Hardware One serves the dashboard at its IP address in a browser. Log in with your admin username and password. The dashboard is the first page you see.

Q: What is the Files tab in the web UI?
A: The filesystem browser is in the Files tab of the web UI. You can browse folders, view text files, download files, upload new files, and delete files from the browser.

Q: How do I upload a file to the device?
A: Files are uploaded through the Files tab in the web UI. Click the upload button, select a file, and choose the destination folder.

Q: How do I update the firmware over the web?
A: OTA updates are done from the Settings tab in the web UI. Upload a new firmware binary to flash it wirelessly. The device reboots automatically after a successful update.

Q: What is OTA?
A: OTA stands for over-the-air update. Hardware One supports flashing new firmware from the web UI without a USB cable. The device validates the binary before writing it to flash.

Q: How do I roll back firmware?
A: Firmware updates keep the previous image in a secondary OTA partition. If the new firmware fails to boot, it reverts automatically. You can also trigger a manual rollback from the web UI settings page.

Q: What is the CLI?
A: The CLI is the command-line interface for Hardware One. You can access it over USB serial at 115200 baud or in the browser via the CLI tab in the web UI.

Q: How do I get a list of all CLI commands?
A: The CLI help system is accessed by typing help. Use a module name like wifi or sensors to see that module's commands. Use help all to print every available command.

Q: How do I use command history in the CLI?
A: The CLI supports command history using the up and down arrow keys to scroll through previously entered commands. Use clear to clear the screen.

Q: What is the build configuration system?
A: Hardware One uses a single header file called System_BuildConfig.h to enable or disable features at compile time. Set a flag to 1 to include a feature and rebuild.

Q: Which features can be enabled in the build?
A: The build system uses feature flags in System_BuildConfig.h. Set a flag to 1 to include a feature, then rebuild and reflash. Each subsystem can be independently enabled or disabled.

Q: What is the difference between Basic and Advanced setup?
A: Hardware One offers two setup modes. Basic asks only for a network name, password, and admin account. Advanced walks through every configurable option.

Q: How do I redo the setup wizard?
A: Hardware One lets you rerun the setup wizard at any time by typing wizard. This lets you reconfigure credentials and other settings without resetting the device.

Q: How do I change the device timezone?
A: Hardware One uses POSIX timezone strings. Set it with settings set timezone followed by the string. For example, settings set timezone EST5EDT for US Eastern time.

Q: How do I see the current time?
A: Hardware One displays the current date and time when you type time. If connected to the internet, the time is synchronised from NTP automatically.

Q: What NTP servers does Hardware One use?
A: Hardware One uses pool.ntp.org by default. You can change the server with settings set ntpServer followed by the server address.

Q: How do I set a static IP address?
A: WiFi can use a static IP by running settings set staticIP followed by the IP address. Also set gateway and subnet. Reconnect for the change to take effect. Leave staticIP empty to use DHCP.

Q: Does Hardware One support WPA2 enterprise WiFi?
A: WiFi on Hardware One supports standard WPA2 personal networks only. Use a personal hotspot or access point for enterprise environments.

Q: What is the maximum WiFi range?
A: WiFi range on Hardware One uses the built-in ESP32 antenna. Typical indoor range is twenty to thirty metres depending on obstacles.

Q: How do I check the WiFi signal strength?
A: WiFi signal strength is shown by typing wifi to see the RSSI value. Higher negative numbers like minus ninety mean weaker signal. Minus sixty or better is generally reliable.

Q: Can Hardware One connect to multiple WiFi networks?
A: WiFi on Hardware One stores one primary SSID and password. It connects automatically on boot. You can change credentials with setssid and setpass at any time.

Q: What MQTT QoS levels are supported?
A: MQTT on Hardware One publishes and subscribes at QoS level 0 by default. This means at-most-once delivery with no acknowledgement.

Q: How do I publish a message to MQTT manually?
A: MQTT messages are published with the mqtt publish command followed by the topic and message. For example, mqtt publish devices/sensor1/test hello.

Q: How do I subscribe to an MQTT topic?
A: MQTT subscriptions are created with the mqtt subscribe command followed by the topic pattern. Use a wildcard like devices/# to subscribe to all sub-topics.

Q: Can Hardware One receive commands over MQTT?
A: MQTT can deliver commands by subscribing to the command topic prefix followed by /cmd. Any CLI command sent as a message to that topic is executed on the device.

Q: How do automations get triggered?
A: Automations trigger in three ways. SCHEDULE with BOOT runs once on startup. SCHEDULE with TIME=HH:MM runs daily at that time. SCHEDULE with INTERVAL=Xm runs repeatedly. Conditional automations with IF run whenever the condition becomes true.

Q: What conditions can I use in automations?
A: Automations support conditions including sensor thresholds like TEMP greater than 75, presence like MOTION=true, zone like ZONE=Upstairs, and time ranges. Combine conditions with AND.

Q: Can automations call other automations?
A: Automations can chain together using the automation run command followed by the name. This allows reusable sub-routines.

Q: How do I create an automation from the web UI?
A: The automation editor is in the Automations tab of the web UI. Click New, give it a name, write the schedule or condition line, then add the commands to execute.

Q: How do I see if an automation ran successfully?
A: Automations log their execution history. Use automation log followed by the name to see timestamps, whether it triggered, and whether each command succeeded.

Q: What is the remote command feature in bonded mode?
A: ESP-NOW bonded mode gives the controller device a Remote tab in the web UI. Commands typed there are sent to and executed on the bonded device.

Q: How do I see what the bonded device is doing?
A: ESP-NOW bonded mode forwards sensor readings and status from the bonded device to the controller. The dashboard shows a summary panel for the bonded device.

Q: How does encrypted ESP-NOW pairing work?
A: ESP-NOW pairing uses an encrypted exchange where both devices verify a hashed version of the shared passphrase. After verification, all messages between them are encrypted.

Q: How many devices can be paired with ESP-NOW?
A: ESP-NOW supports up to six paired peers. Each peer is identified by its MAC address and given a name and room assignment.

Q: What is metadata sync?
A: ESP-NOW metadata sync periodically broadcasts device name, zone, room, uptime, and sensor summary to all paired peers. This lets the network maintain a live picture of every device without a central server.

Q: How do I see metadata from peers?
A: ESP-NOW peers and their metadata are listed with the espnow peers command. In the web UI, the ESP-NOW tab shows a card for each peer with its current status.

Q: What is the NeoPixel LED used for?
A: The NeoPixel RGB LED can show device status, alert conditions, or decorative colours. Automations can set the LED colour based on sensor readings or events.

Q: What colours can the LED be set to?
A: NeoPixel colours are set with the ledcolor command followed by a colour name like red, green, blue, white, orange, yellow, purple, or cyan. Use ledcolor off to turn it off.

Q: How do I set the LED to a specific RGB value?
A: NeoPixel RGB values are set with the ledrgb command followed by three numbers for red, green, and blue each from 0 to 255. For example, ledrgb 255 128 0 sets an orange colour.

Q: How do I set LED brightness?
A: NeoPixel brightness is set with the ledbrightness command followed by a value from 0 to 255. This scales all colour output without changing the hue.

Q: What is the Seesaw gamepad?
A: The Seesaw is an Adafruit I2C gamepad and joystick. On Standard Handheld builds it is used to navigate the OLED menu and control device functions.

Q: How do I navigate the OLED menu with the gamepad?
A: The OLED menu is navigated with the joystick to move the cursor. Press the A button to select. Press B to go back. Long-press A to enter edit mode for a setting.

Q: What does the OLED display show when idle?
A: The OLED shows a status screen with connectivity state, IP address, sensor count, and uptime when idle. Press any button on the gamepad or type oled menu to open the interactive menu.

Q: How do I set a custom OLED screen message?
A: The OLED display shows custom text when you use the oled text command followed by the message. It stays until the next status update or until you use oled clear.

Q: What is the barebones headless configuration?
A: Hardware One in barebones headless mode runs on an ESP32-S3 without any display or gamepad. All interaction is through the CLI over serial or the web UI.

Q: What is the sensor appliance configuration?
A: Hardware One in sensor appliance mode enables multiple sensors and MQTT publishing but no OLED or gamepad. It is designed for fixed installations that report data to a home automation server.

Q: How do I change the hostname?
A: Hardware One hostname is set with settings set hostname followed by the name. The hostname appears in the serial monitor on boot and is used for mDNS.

Q: What is mDNS?
A: Hardware One supports mDNS, which lets you reach the device using a name like hardwareone.local instead of an IP address. The name is set with the hostname setting.

Q: How do I see the device MAC address?
A: Hardware One has separate MAC addresses for each radio interface. Each subsystem command shows its own address. Use the relevant status command for the interface you need.

Q: What is the difference between WiFi and ESP-NOW MAC addresses?
A: Hardware One uses different MAC addresses for different radio interfaces. When pairing devices, use the MAC address shown in espnow status, not the station address.

Q: How do I enable verbose logging for a sensor?
A: Debug logging for sensors is enabled with the debug command followed by the sensor name and 1. For example, debugsensors 1 enables general sensor debug output. Add temp to enable only until the next reboot.

Q: How do I see a list of all debug flags?
A: Debug flags are listed with the debug list command. It shows all available flags and their current state. Flags with a 1 are enabled and flags with a 0 are disabled.

Q: Can I log debug output to a file?
A: Debug output is redirected to a file with debugfile on, which writes to /data/logs/debug.txt. Use debugfile off to stop. Monitor file size with fsusage.

Q: What is the auth system?
A: Hardware One uses a username and password system for the web UI and CLI access. The first account created during setup is the admin. Admin accounts can manage other users.

Q: What happens if I forget the admin password?
A: Hardware One can be factory reset with settings reset from serial access. This erases all settings including user accounts. Then run the setup wizard again. Back up first with settings export.

Q: How do I import settings?
A: Hardware One settings are imported with settings import followed by a JSON string. The format matches the output of settings export. Reboot after importing for all settings to take effect.

Q: What does the status command show?
A: Hardware One shows a summary of the current device state when you type status. It includes connectivity, active sensors, uptime, and memory usage.

Q: How do I check if a feature is enabled?
A: Hardware One shows active modules when you type status. You can also type help followed by the module name to check if its commands are available.

Q: What happens when memory is low?
A: Memory warnings are logged when free memory falls below a threshold. Use memory or memreport to see a full breakdown. Consider disabling unused sensors if memory is critically low.

Q: How do I reduce memory usage?
A: Memory can be freed by disabling sensors you are not using with the closeSENSOR command. Reduce the LLM context length in settings if using the language model. Use memreport to identify which components use the most.

Q: What is PSRAM?
A: Memory on the ESP32-S3 includes PSRAM, an external pseudo-static RAM chip. Hardware One uses it for large allocations like LLM weights, KV cache, and sensor buffers. Supported boards have 8 MB of PSRAM.

Q: What is the difference between heap and PSRAM?
A: Memory is split between internal heap and PSRAM. The internal heap is fast SRAM built into the ESP32 chip for time-critical data. PSRAM is larger but slower, accessed over SPI, and used for large allocations.

Q: How do I reboot into safe mode?
A: Hardware One enters safe mode when you hold the boot button during power-on. In safe mode, automations do not run and only basic commands are available.

Q: What is the boot log?
A: Hardware One prints a detailed boot sequence to serial at 115200 baud. It lists each module as it initialises, reports any failures, and shows the available memory after setup.

Q: How do I see previous boot logs?
A: Hardware One does not store boot logs by default. Connect serial before powering on to capture the boot sequence. Enable debugfile on before rebooting for persistent logging.

Q: What happens during a firmware panic?
A: Hardware One prints a stack trace to serial and reboots if the firmware crashes. Serial monitor at 115200 baud shows the crash address. Use idf.py monitor to decode addresses into function names.

Q: How do I decode a crash stack trace?
A: Debug stack traces are decoded by running idf.py monitor while connected via USB. Alternatively use xtensa-esp32s3-elf-addr2line with the ELF file from your build directory.

Q: What is the difference between reboot and reset?
A: Hardware One distinguishes between reboot and reset. Reboot restarts the firmware and preserves all settings. Settings reset restores factory defaults and erases all user configuration.

Q: How do I update only one setting without changing others?
A: Hardware One settings are updated individually with settings set followed by the key and value. Other settings are not affected. Use settings list to find the exact key name.

Q: What format do settings use?
A: Hardware One stores settings as JSON in the internal filesystem. The settings export command outputs all current settings as a JSON object for backup and restore.

Q: How does Hardware One handle power loss?
A: Hardware One uses LittleFS which is power-safe and will not corrupt data during a power cut. Settings and automations are preserved. Sensor logs may have an incomplete final entry.

Q: Can Hardware One run on battery power?
A: Battery power is supported on any board with a LiPo connector. Use battery status to see voltage, charge percentage, and charging state. Battery monitoring must be enabled in System_BuildConfig.h.

Q: What voltage does the battery connector expect?
A: Battery input expects a 3.7 volt single-cell LiPo battery. The board's built-in charger handles charging from USB. Do not exceed 4.2 volts.

Q: How do I put the device to sleep to save power?
A: Hardware One enters deep sleep with the sleep command followed by the number of seconds. The device wakes after that time and resumes normal operation.

Q: Does the RTC retain time during sleep?
A: Hardware One uses a DS3231 RTC with its own battery backup that keeps time independently. After waking from sleep, the firmware reads the current time from the RTC automatically.

Q: What is the difference between opengps and gpsauto?
A: GPS has two start modes. opengps starts the module manually for the current session. gpsautostart on makes it start automatically on every boot.

Q: How do I calibrate the IMU?
A: The IMU calibration is started with the imucal command. Follow the on-screen instructions to rotate the device through different orientations. Calibration data is saved and loaded automatically on the next boot.

Q: What is the presence sensor range?
A: The STHS34PF80 IR presence sensor detects human presence and motion at up to four metres in a roughly ninety degree cone. It is passive infrared so it responds to body heat.

Q: What are the APDS9960 gesture directions?
A: The APDS9960 in gesture mode detects swipes in four directions: up, down, left, and right. Use apdsgesture to start the gesture reader. Results can trigger automations.

Q: How accurate is the thermal camera?
A: The MLX90640 has temperature accuracy of about one to two degrees Celsius across its 32 by 24 pixel array. It measures surface temperatures from negative forty to three hundred degrees Celsius.

Q: What is the ToF sensor maximum range?
A: The VL53L4CX measures distances from one centimetre to six metres with millimetre precision. It supports up to four simultaneous ranging targets.

Q: How do I use the servo controller with more than one servo?
A: The PCA9685 I2C PWM controller supports sixteen independent channels. Use the servo command followed by the channel number from 0 to 15 and an angle from 0 to 180.

Q: What I2C address does the PCA9685 use?
A: The PCA9685 default I2C address is 0x40. If you have multiple boards, set their address pins to get different addresses and configure in System_BuildConfig.h.

Q: What is the FM radio frequency range?
A: The RDA5807 FM radio covers 76 to 108 MHz. Use fmradio tune followed by the frequency in MHz. Use fmradio volume followed by a value from 0 to 15 to set the volume.

Q: Can I record audio with the FM radio?
A: The RDA5807 is a receive-only FM radio. It plays audio through a connected speaker or headphones. Recording is not supported.

Q: What is the web UI built with?
A: Hardware One serves a single-page web application from flash. It uses plain HTML, CSS, and JavaScript with no external dependencies and works in any modern browser.

Q: How many simultaneous web UI connections are supported?
A: Hardware One supports up to four simultaneous HTTP connections. Only one user at a time should use the CLI tab to avoid command conflicts.

Q: What port does the web server run on?
A: Hardware One runs the web server on port 80 by default. You can change this with settings set webPort followed by the port number.

Q: Is the web UI accessible over HTTPS?
A: Hardware One serves the web UI over plain HTTP. For secure access, use a reverse proxy or VPN on your local network.

Q: What is the Pair tab?
A: ESP-NOW pairing is done in the Pair tab of the web UI. Enter a shared passphrase on both devices, then click Pair on one and Accept on the other.

Q: What is the MQTT tab in the web UI?
A: MQTT status is shown in the MQTT tab of the web UI. It displays the broker connection status, topic prefix, and a log of recent published and received messages.

Q: How do I see the ESP-NOW tab?
A: ESP-NOW peers are shown in the ESP-NOW tab of the web UI. It shows a card for each paired peer with its name, room, last-seen time, and recent metadata.

Q: What is the Remote tab?
A: ESP-NOW bonded mode provides a Remote tab on the controller device. It gives a unified view of the bonded device including its sensors, status, and a CLI that routes commands to it.

Q: What is the Settings tab in the web UI?
A: Hardware One settings are editable in the Settings tab of the web UI. Changes save immediately. It also has sections for user management, OTA firmware update, and factory reset.

Q: What does the Sensors tab show?
A: The Sensors tab shows live readings from all active sensors, the I2C device table, sensor log status, and controls to start or stop individual sensors. Data updates every two seconds.

Q: How do I write an automation that responds to temperature?
A: Automations can respond to temperature with IF TEMP>threshold THEN command. For example, IF TEMP>30 THEN ledcolor red sets the LED red when temperature exceeds 30 degrees.

Q: Can automations send messages to MQTT?
A: Automations can publish MQTT messages by including mqtt publish followed by the topic and message as a command. For example, IF TEMP>80 THEN mqtt publish alerts/temp too hot.

Q: How do I write a multi-command automation?
A: Automations support multiple commands separated by a semicolon. For example: ledcolor red; mqtt publish alerts/motion detected executes both commands in order when triggered.

Q: What does help all output?
A: The CLI prints a full list of every registered command across all modules when you type help all. Each line shows the command name and a brief description.

Q: How do I use the interactive help system?
A: The CLI help mode is entered by typing help. Then type a module name to see its commands. Type a command name for detailed usage. Type exit to leave help mode.

Q: What happens if I type an unknown command?
A: The CLI prints unknown command and shows the closest matching command name if one exists. Use help followed by the module name to see the correct syntax.

Q: How do I comment out a line in an automation?
A: Automations support comments by starting a line with a hash symbol. Comment lines are ignored during execution.

Q: Can Hardware One act as a WiFi access point?
A: WiFi access point mode is started with wifi ap on. Other devices can connect directly to it. The default SSID is the device hostname. Set a password with wifi appass.

Q: How do I disable the access point?
A: WiFi access point is stopped with wifi ap off, returning the device to station-only mode.

Q: Can Hardware One be both an access point and a station at the same time?
A: WiFi supports AP-STA mode where the device connects to your home network while also broadcasting its own access point. Enable it with wifi ap on while already connected.

Q: What firmware framework does Hardware One use?
A: Hardware One runs on ESP-IDF, the official Espressif IoT Development Framework. It does not use the Arduino IDE or Arduino libraries.

Q: Why does Hardware One use ESP-IDF instead of Arduino?
A: Hardware One uses ESP-IDF because it provides better control over memory, task scheduling, and peripherals. It allows true multitasking with FreeRTOS tasks running concurrently.

Q: What tasks run on Hardware One?
A: Hardware One runs multiple FreeRTOS tasks including the CLI handler, network managers, the web server, sensor polling loops, the automation engine, and the LLM inference task when active.

Q: Can I add custom commands to the CLI?
A: The CLI supports custom commands by adding an entry to the command table. Specify the name, description, argument format, and handler function. The command appears in help after rebuilding.

Q: What is the command registry?
A: The CLI command registry is a table of all registered commands. Commands are registered at boot time and appear automatically in the help system.

Q: How do I check the firmware version?
A: Hardware One displays the firmware version when you type version. It shows the version string, build date, and target board.

Q: What does the version command show?
A: Hardware One outputs the firmware version number, the git commit hash if available, the build timestamp, and the target hardware configuration.

Q: Can I run Hardware One without PSRAM?
A: Hardware One basic features work without PSRAM on the ESP32-S3. However, the LLM requires PSRAM for weight storage and the KV cache.

Q: How do I reduce flash usage?
A: Hardware One flash usage is reduced by disabling unused features in System_BuildConfig.h before building. Use idf.py size to see the flash footprint of each component.

Q: What is the typical flash usage?
A: Hardware One with all features enabled uses approximately one to one and a half megabytes of program flash. The remaining flash is used for LittleFS storage.

Q: How big is the LittleFS partition?
A: The filesystem partition is defined in the partition table. A typical configuration allocates two to three megabytes for internal storage. Check partitions.csv for your build's layout.

Q: How do I check the partition layout?
A: Hardware One partition layout is shown by running idf.py partition-table. It displays the name, type, offset, and size of each partition.
