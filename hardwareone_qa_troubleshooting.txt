Q: My WiFi keeps disconnecting. What should I do?
A: WiFi disconnections are usually caused by 5GHz networks. Hardware One only supports 2.4GHz. Run wifistatus to check signal strength. If RSSI is below -80, move the device closer to the router.

Q: WiFi says connected but I cannot reach the web interface.
A: WiFi may be connected but unreachable if your browser device is on a different network. Run ipinfo to check the assigned IP address and try accessing it directly in your browser.

Q: WiFi will not connect to my network.
A: WiFi on Hardware One only supports 2.4GHz networks. Check your SSID and password with settings. Run wifiscan to verify your network is visible.

Q: How do I reset WiFi settings?
A: WiFi credentials are erased with the wificlear command. The device will restart in setup mode with the access point enabled.

Q: A sensor is not detected on I2C.
A: I2C devices can be checked with i2cscan to see which addresses respond. Verify SDA and SCL wiring and pull-up resistors. If the bus shows no devices, check power to the sensor board.

Q: The BME280 shows incorrect temperature readings.
A: The BME280 can self-heat when read frequently. Make sure it has airflow. Temperature offset can be adjusted in settings. Run bme to see raw readings.

Q: The IMU sensor shows all zeros.
A: The IMU may need recalibration. Run i2cscan to check if the ICM-42670-P responds at its address. If present but reading zeros, run imucal to recalibrate. The IMU must be stationary during calibration.

Q: The VL53L1X distance sensor gives inconsistent readings.
A: The VL53L1X works best with a clear line of sight between 4cm and 200cm. Transparent or very dark surfaces may not reflect the laser properly. Run tof to see raw distance values.

Q: The LLM model will not load.
A: The LLM requires at least 7MB free PSRAM. Check with meminfo. Run llmstatus for the specific error. Common causes are file not found, file corrupted, or insufficient PSRAM. Try a lower context with llmload model.bin 64.

Q: The LLM model loads but produces garbage output.
A: The LLM model file may be corrupted or incompatible. Re-convert it using the converter with INT8 format. Check that the model was trained with the correct tokenizer. Run debugllmgenerate 1 for detailed sampling logs.

Q: The LLM is very slow.
A: The LLM speed depends on model size. Run llmstatus to check tokens per second. Normal speed is 2-8 tok/s. Lower context length reduces KV cache and speeds up inference.

Q: MQTT will not connect to my broker.
A: MQTT connection issues can be diagnosed with mqttstatus. Verify the broker address, port, username, and password with settings. The default port is 1883. A network connection must be active first.

Q: MQTT connects but no data appears on my broker.
A: MQTT publishing uses the configured topic prefix. Check it with settings. Use a tool like MQTT Explorer to subscribe to the wildcard topic. Run debugmqtt 1 for publish logs.

Q: How do I use Hardware One with Home Assistant?
A: MQTT integration with Home Assistant requires configuring the MQTT broker address in Hardware One settings. Sensor data is published as MQTT topics matching the configured topic prefix.

Q: ESP-NOW is not receiving messages from other devices.
A: ESP-NOW requires both devices to be on the same radio channel. Run espnowstatus to check. Add the peer MAC address with espnowadd. Run debugespnow 1 to see incoming packets.

Q: How many sensors can I connect?
A: I2C supports multiple sensors simultaneously. The practical limit is around 8-12 devices before bus capacitance becomes an issue. Each sensor uses a unique I2C address. Run i2cscan to see all detected devices.

Q: What is the range of ESP-NOW?
A: ESP-NOW range is typically 10-30 meters indoors through walls, and up to 100 meters outdoors with line of sight. Range depends on obstacles, interference, and antenna orientation.

Q: How long does the battery last?
A: Battery life depends on which features are active. With sensors running, expect 4-8 hours on a typical LiPo battery. Reducing sensor polling extends battery life significantly. Run battery to check current voltage.

Q: How do I factory reset the device?
A: Hardware One is factory reset with the factoryreset command from the CLI. This clears all settings, credentials, and user accounts. The device restarts in first-time setup mode.

Q: The OLED display is blank or frozen.
A: The OLED can be tested with oledtest to verify it works. Check the I2C connection with i2cscan. If the display appears on the scan but is blank, run oledclear then oledrefresh. A frozen display may indicate a bus lockup.

Q: The device keeps restarting in a loop.
A: Hardware One restart loops are usually caused by a crash or watchdog timeout. Connect via USB serial to see the crash log. Common causes are I2C bus lockup, out of memory, or corrupted settings. Hold the boot button during startup for safe mode.

Q: How do I check memory usage?
A: Memory usage is shown with meminfo, which displays free heap, PSRAM, and stack usage. Run taskinfo to see per-task stack usage. If free heap is below 30KB the device may become unstable.

Q: OTA firmware update fails.
A: OTA updates require a stable connection during the update. The firmware binary must fit in the OTA partition. Run otastatus to check the update state. A failed update rolls back to the previous firmware automatically.

Q: How do I enable debug logging?
A: Debug logging is enabled per category. Use debug followed by the module name and 1 to enable it. Add temp to make it temporary until reboot. Run debugstatus to see active categories.

Q: How do I access the CLI?
A: The CLI is accessed over USB serial at 115200 baud using a terminal program like PuTTY or screen. Type help to see all available commands. The web terminal also provides CLI access.

Q: The web interface is very slow.
A: WiFi signal strength affects web UI performance. Check with wifistatus. The web server runs on a single thread so concurrent requests queue up. Reduce polling intervals if multiple browsers are connected.

Q: Can I use Hardware One outdoors?
A: Hardware One is not waterproof and requires a weatherproof enclosure for outdoor use. Operating temperature range is -10C to 60C. Direct sunlight may affect temperature sensor readings.

Q: How do I update the firmware?
A: OTA firmware updates are done by uploading the binary through the web interface OTA page, or using the otaupdate CLI command. The device verifies the binary, writes it, and reboots.

Q: What board variants does Hardware One support?
A: Hardware One runs on ESP32-S3 boards with PSRAM. The primary target is the ESP32-S3-DevKitC-1 with 8MB PSRAM.

Q: Can multiple Hardware One devices talk to each other?
A: ESP-NOW enables direct peer-to-peer communication between Hardware One devices without a router. Both devices must be added as peers with their MAC addresses.
